# ポートフォリオサイト デザイン要件書

## 概要

エンジニア個人のポートフォリオサイトのリニューアル。
制作物・ブログ・プロフィールを掲載し、自分を知ってもらうためのサイト。

---

## デザインの方向性

- **カラー**: 白ベース × 水色のアクセント（現サイトの路線を踏襲）
- **雰囲気**: kawaii、柔らかい、親しみやすい
- **情報量**: 少なめ。余白を活かしてスッキリ見せる
- **アニメーション**: スクロールに連動したフェードイン、ホバーエフェクトなど動きのあるサイトにしたい

---

## ページ構成

### トップページ（1ページ、セクション分割）

以下の順で縦スクロール。

#### 1. Hero

| 要素           | 内容                                            |
| -------------- | ----------------------------------------------- |
| アイコン画像   | プロフィール画像（丸型想定）                    |
| 名前           | もっくん                                        |
| キャッチコピー | 短い一文（内容は後で確定）                      |
| ナビボタン     | Profile / Works / Blog へスクロールするボタン群 |

- ナビボタンは浮遊するカード風のデザインを検討中（提案歓迎）
- 背景にアニメーション演出が欲しい（現サイトでは浮遊する四角の背景あり）
- ファーストビューで訪問者が各コンテンツにすぐアクセスできることが目的

#### 2. Profile

| 要素         | 内容                                     |
| ------------ | ---------------------------------------- |
| 自己紹介文   | 短めの文章で「何をしている人か」を伝える |
| 経歴         | 文章形式（年表UIではない）               |
| 技術スタック | 3カテゴリに分類して表示                  |

- 技術スタックの分類:
  - **Frontend**: HTML, CSS, JavaScript, TypeScript, React, Next.js など
  - **Backend**: Node.js, Express, Python, Java など
  - **Tools**: Git/GitHub, Firebase, Supabase, Vercel など
- 各スタックは **アイコン + 名前** のみ（レベル表記なし）
- タグ風 or バッジ風の並びをイメージ

#### 3. Works

| 要素             | 内容                             |
| ---------------- | -------------------------------- |
| 制作物カード一覧 | サムネイル + タイトル + 短い説明 |

- カードをクリックすると詳細ページへ遷移
- カードのホバーエフェクトが欲しい

#### 4. Blog

| 要素           | 内容                                         |
| -------------- | -------------------------------------------- |
| 記事カード一覧 | サムネイル or アイキャッチ + タイトル + 日付 |

- カードをクリックすると記事詳細ページへ遷移
- 最新数件の表示 +「もっと見る」で一覧ページへ

#### 5. Footer

| 要素           | 内容                     |
| -------------- | ------------------------ |
| SNSリンク      | Twitter(X), GitHub, Zenn |
| メールアドレス | 連絡先                   |
| コピーライト   | (c) もっくん             |

- コンパクトにまとめる

---

### 詳細ページ

#### Works 詳細（/works/[slug]）

| 要素                            | 内容                     |
| ------------------------------- | ------------------------ |
| タイトル                        | 制作物の名前             |
| サムネイル / スクリーンショット | メイン画像               |
| 説明文                          | 概要・工夫した点など     |
| 使用技術                        | タグ形式                 |
| リンク                          | GitHub / デモサイト など |

#### Blog 記事（/blog/[slug]）

| 要素     | 内容                            |
| -------- | ------------------------------- |
| タイトル | 記事タイトル                    |
| 投稿日   | 日付                            |
| 本文     | Markdown から生成される記事本文 |
| タグ     | カテゴリ・技術タグ              |

- 記事本文のタイポグラフィ（見出し・コードブロック・リスト等）のスタイルも必要

#### Blog 一覧（/blog）

| 要素              | 内容                                       |
| ----------------- | ------------------------------------------ |
| 記事カード一覧    | トップの Blog セクションと同様だが全件表示 |
| フィルター / 検索 | 必要に応じて（任意）                       |

## アニメーション

| 種類                   | 適用箇所                                 |
| ---------------------- | ---------------------------------------- |
| スクロールフェードイン | 各セクションの要素が画面内に入ったら表示 |
| ホバーエフェクト       | カード類、ボタン類                       |
| ページ遷移             | View Transitions API で滑らかな画面遷移  |
| Hero背景               | 浮遊アニメーション等の演出               |
